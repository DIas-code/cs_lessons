These are solved together as a class. Instructor codes live on the projector, students follow along and suggest the next step.

Exercise 1 — Student Grade Report Program.cs 
// Full program: read N student scores, compute stats, print a report
// Covers: arrays, for, foreach, methods, string formatting

int[] scores = ReadScores();
PrintReport(scores);

// ------------------------------------------------

static int[] ReadScores()
{
    Console.Write("Number of students: ");
    int n = int.Parse(Console.ReadLine()!);
    int[] arr = new int[n];
    for (int i = 0; i < n; i++)
    {
        Console.Write($"  Score [{i + 1}]: ");
        arr[i] = int.Parse(Console.ReadLine()!);
    }
    return arr;
}

static void PrintReport(int[] scores)
{
    int sum = 0, passing = 0, min = scores[0], max = scores[0];
    foreach (int s in scores)
    {
        sum += s;
        if (s >= 60) passing++;
        if (s < min) min = s;
        if (s > max) max = s;
    }
    double avg = (double)sum / scores.Length;

    Console.WriteLine("\n=== Grade Report ===");
    Console.WriteLine($"Students : {scores.Length}");
    Console.WriteLine($"Average  : {avg:F1}");
    Console.WriteLine($"Highest  : {max}");
    Console.WriteLine($"Lowest   : {min}");
    Console.WriteLine($"Passing  : {passing}/{scores.Length}");
}
Exercise 2 — Bubble Sort (manual implementation) Program.cs 
// Implement bubble sort without Array.Sort()
// This shows HOW sorting works internally

static void BubbleSort(int[] arr)
{
    int n = arr.Length;
    for (int i = 0; i < n - 1; i++)
    {
        for (int j = 0; j < n - 1 - i; j++)
        {
            if (arr[j] > arr[j + 1])
            {
                // Swap adjacent elements
                int temp = arr[j];
                arr[j]     = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

int[] data = { 64, 34, 25, 12, 22, 11, 90 };
Console.WriteLine("Before: " + string.Join(", ", data));
BubbleSort(data);
Console.WriteLine("After:  " + string.Join(", ", data));
Exercise 3 — Linear Search Program.cs 
// Find first and ALL occurrences of a value in an array

static int FindFirst(int[] arr, int target)
{
    for (int i = 0; i < arr.Length; i++)
        if (arr[i] == target) return i;   // found — return index
    return -1;                            // not found
}

static int[] FindAll(int[] arr, int target)
{
    // Count first so we know the result array size
    int count = 0;
    foreach (int n in arr) if (n == target) count++;

    int[] result = new int[count];
    int idx = 0;
    for (int i = 0; i < arr.Length; i++)
        if (arr[i] == target) result[idx++] = i;
    return result;
}

int[] nums = { 4, 7, 2, 7, 9, 7, 1 };
Console.WriteLine(FindFirst(nums, 7));                        // 1
Console.WriteLine(string.Join(", ", FindAll(nums, 7)));       // 1, 3, 5

✏️ Independent Practice Tasks

Students work independently or in pairs for 25–30 minutes. Instructor circulates and helps.

Task 1 Palindrome Checker Medium

Write static bool IsPalindrome(string s) that returns true if the string reads the same forwards and backwards (e.g., "racecar", "level").

  • Compare characters from both ends moving towards the centre
  • Ignore case: "Racecar" should also return true
  • Test with at least 5 strings
Task 2 Two-Array Merge Medium

Write static int[] Merge(int[] a, int[] b) that returns a new array containing all elements from both arrays, sorted ascending.

  • Create a result array of length a.Length + b.Length
  • Copy both arrays into it using Array.Copy()
  • Sort the result with Array.Sort()
  • Return the result — do not modify a or b
Task 3 Number Frequency Counter Hard

Given an array of integers (values 1–10), count how many times each value appears.

  • Use a second array of size 11 as a frequency counter: freq[value]++
  • Print only values that appear at least once
  • Example: input {3,1,3,2,1,1} → print:
    1 appears 3 times
    2 appears 1 time
    3 appears 2 times

🐛 Debugger Walkthrough

⚡ Live demo — Visual Studio 2026 Debugger
  • Set a breakpoint — click the grey margin left of a line number (red dot appears). The program pauses there when running with F5.
  • Step Over (F10) — execute the current line and move to the next, without entering called methods.
  • Step Into (F11) — if the current line calls a method, jump into it and pause at its first line.
  • Watch window — hover over a variable to see its current value, or add it to the Watch window to track it across steps.
  • Demo suggestion: Set a breakpoint inside the BubbleSort inner loop. Step through one full pass, watching arr[j] and arr[j+1] change in the Watch window. Students immediately understand the swap logic visually.

⚠️ Patterns to Watch For

Giant Main with no methods
After two lessons on methods, students still write 60-line Main functions. If you see this in code review, stop and refactor it together — point to the Single Responsibility principle and ask: "what does this block of 20 lines do? Give it a name. That's your method."
Duplicated logic instead of calling the method again
A student writes the same loop twice in two different places instead of extracting it into a method and calling it twice. Highlight these when reviewing — duplication is the most common source of bugs because changes must be made in multiple places.
Using Console.WriteLine inside utility methods
A method like static int FindMax(int[] arr) should return the max, not print it. Printing is the caller's job. Methods that mix logic and output are impossible to reuse. Separate computation from output.
