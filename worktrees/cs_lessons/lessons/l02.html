Part 1 — C# Basics

L02 — Variables, Constants & Data Types

13 February 2026 · Friday · 80 min Done

We learn how to store data in a program using variables and constants. We explore all the fundamental data types in C#, understand the difference between value types and reference types, and practice type conversion.

🎯 Learning Objectives

  • Declare, initialize, and reassign variables using correct C# syntax
  • Name and use the most common C# data types: int, double, decimal, bool, char, string
  • Explain the difference between value types and reference types
  • Declare constants with const and understand when to use them
  • Convert between types using implicit conversion, explicit cast, and the Convert class
  • Use var for local type inference and know when it is appropriate

📖 Theory

1. Variables

A variable is a named container in memory that stores a value. Every variable has:

  • A type — defines what kind of data it holds
  • A name — how you refer to it in your code
  • A value — the data it currently holds

The general syntax to declare a variable:

Syntax 
// Declaration + initialization in one line (recommended)
type variableName = value;

// Declaration first, then assignment later
type variableName;
variableName = value;

Naming rules and conventions:

  • Must start with a letter or underscore (_), not a digit
  • Can contain letters, digits, underscores — no spaces or special characters
  • C# is case-sensitive: age and Age are different variables
  • By convention, local variables use camelCase: firstName, totalPrice
  • Avoid single letters (except i, j in loops); use descriptive names
Good naming = self-documenting code. A variable named n tells you nothing. A variable named studentCount tells you exactly what it holds. Write code that your future self (and teammates) can read.

2. C# Data Types

C# is a strongly typed language — every variable has a fixed type that cannot change. The type determines: what values are valid, how much memory is used, and what operations are allowed.

Integer types (whole numbers — no decimal part):

Type Size Range When to use
byte 1 byte 0 to 255 Small positive integers, pixel values
short 2 bytes −32,768 to 32,767 Rarely used; memory-critical contexts
int 4 bytes −2.1 billion to 2.1 billion Default choice for integers
long 8 bytes ≈ −9.2 × 10¹⁸ to 9.2 × 10¹⁸ Very large numbers (file sizes, IDs)

Floating-point types (numbers with a decimal point):

Type Size Precision When to use
float 4 bytes ~7 digits Graphics, game physics (append f: 3.14f)
double 8 bytes ~15 digits Default for most decimal math
decimal 16 bytes 28–29 digits Money, financial calculations (append m: 9.99m)
Use decimal for money — always. float and double use binary floating-point, which cannot represent 0.1 exactly. Try 0.1f + 0.2f — you get 0.3000000119..., not 0.3. For financial calculations this is unacceptable. decimal uses base-10 arithmetic and avoids this problem.

Other essential types:

Type Description Example literals
bool Boolean — true or false only true, false
char A single Unicode character 'A', 'z', '5', '!' (single quotes)
string A sequence of characters (text) "Hello", "Alice", "" (double quotes)
object Base type — can hold any value Advanced; avoid for now

3. Value Types vs Reference Types

This is one of the most important conceptual distinctions in C#:

Value types

Store the data directly in the variable. When you assign one value-type variable to another, a copy of the data is made. Changes to one do not affect the other.

Examples: int, double, decimal, bool, char, float, all struct types.

Reference types

Store a reference (pointer) to the data in memory. When you assign one reference-type variable to another, both variables point to the same object.

Examples: string, arrays, classes, object.

Note on strings: string is technically a reference type, but it behaves like a value type in most situations because strings in C# are immutable — once created, a string value cannot be changed. All string "modifications" create a new string. You don't need to worry about this distinction now — it becomes important later.

4. Constants — const

A constant is a variable whose value cannot change after it is set. Use constants for values that are fixed by definition: mathematical constants, limits, configuration values.

  • const — value must be known at compile time (a literal or simple expression)
  • Naming convention: PascalCase for constants — MaxScore, Pi
  • Attempting to change a const causes a compile error — not a runtime error
When to use const: Mathematical constants (PI, E), fixed limits (max attempts, array size), unit conversion factors, magic numbers that appear more than once in your code. Using constants instead of raw numbers ("magic numbers") makes your code much easier to read and maintain.

5. Type Inference with var

C# can infer the type of a local variable from the assigned value. The var keyword tells the compiler: "you figure out the type."

Important: var is still strongly typed — the type is fixed at compile time. It is just shorthand to avoid writing the type explicitly when it is obvious.

  • Use var when the type is obvious from the right side: var name = "Alice";
  • Prefer explicit types when clarity matters: decimal price = 9.99m;
  • Never use var for a variable declared without initialisation

6. Type Conversion

Converting a value from one type to another is very common. C# provides several mechanisms:

Implicit conversion (automatic, no data loss)

The compiler allows it automatically when the destination type can hold all possible values of the source type.

  • int → long → double — all safe (larger range)
  • int → double — safe: int x = 5; double d = x;

Explicit cast (you force it, possible data loss)

When converting to a smaller or less precise type, you must explicitly acknowledge the risk with a cast operator (type).

  • double → int: the decimal part is truncated (not rounded): (int)3.9 → 3
  • long → int: safe only if the value fits; otherwise, overflow

Convert class — safe, readable conversions

The Convert class (from the BCL) provides methods for converting between types. Unlike a cast, it rounds floating-point numbers instead of truncating.

  • Convert.ToInt32(), Convert.ToDouble(), Convert.ToString()

Parse and TryParse — from string to number

When you read user input (from the console), it is always a string. To use it as a number you must parse it:

  • int.Parse("42") — converts the string "42" to int 42. Throws an exception if the string is not a valid number.
  • int.TryParse("42", out int result) — safer: returns true/false instead of throwing (covered more in later lessons).

7. Nullable Types

By default, value types (int, double, etc.) cannot hold null. A nullable type is declared by appending ? to the type name, allowing it to also hold null (meaning "no value").

Example: int? score = null; — useful when a value might be "unknown" or "not yet set".

💻 Code Examples

C# — Declaring Variables Program.cs 
// ---- Integer types ----
int    age           = 21;
long   population    = 8_000_000_000L;   // L suffix = long literal; _ for readability
byte   level         = 255;

// ---- Floating-point types ----
float   temperature  = 36.6f;            // f suffix required for float literals
double  pi           = 3.14159265358979;  // default for decimal literals
decimal price        = 19.99m;           // m suffix required for decimal literals

// ---- Other types ----
bool   isStudent     = true;
char   grade         = 'A';              // single quotes for char
string name          = "Alice";          // double quotes for string

// ---- Print them all ----
Console.WriteLine($"Name: {name}, Age: {age}");
Console.WriteLine($"Price: {price} | Is student: {isStudent}");
Console.WriteLine($"Grade: {grade} | Pi: {pi}");
C# — Constants Program.cs 
// Constants — value set once, never changed
const double  Pi          = 3.14159265358979;
const int     MaxAttempts = 3;
const string  AppVersion  = "1.0.0";

double radius = 5.0;
double area   = Pi * radius * radius;

Console.WriteLine($"Circle area: {area:F2}");   // :F2 = 2 decimal places
Console.WriteLine($"Max login attempts: {MaxAttempts}");

// This line would cause a COMPILE ERROR — you cannot change a const:
// Pi = 3.0;   ← Error: "The left-hand side of an assignment must be a variable"
C# — var (Type Inference) Program.cs 
// var — compiler infers the type from the right-hand side
var city     = "Almaty";   // compiler infers: string
var count    = 42;         // compiler infers: int
var ratio    = 0.75;       // compiler infers: double
var isActive = true;       // compiler infers: bool

// Hover over the variable in Visual Studio to see what type was inferred

// These are IDENTICAL — var is just shorthand:
string a = "Hello";
var    b = "Hello";   // same type: string

Console.WriteLine($"{city} | count: {count} | ratio: {ratio}");
C# — Type Conversion Program.cs 
// ---- Implicit conversion (safe, no data loss) ----
int    myInt    = 100;
double myDouble = myInt;    // int → double automatically (100.0)
Console.WriteLine(myDouble); // 100

// ---- Explicit cast (forced, possible data loss) ----
double d = 9.99;
int    i = (int)d;          // Truncates! Result: 9  (NOT rounded)
Console.WriteLine(i);       // 9

// ---- Convert class (rounds, clearer intent) ----
double price   = 9.99;
int    rounded = Convert.ToInt32(price);  // Rounds to nearest: 10
Console.WriteLine(rounded);               // 10

// ---- Parse — string to number ----
string input  = "42";
int    number = int.Parse(input);   // "42" → 42
Console.WriteLine(number + 8);       // 50

// ---- number to string ----
int    score = 95;
string msg   = "Score: " + score;      // int auto-converts to string here
Console.WriteLine(msg);               // "Score: 95"
C# — Floating-Point Precision Demo Program.cs 
// Demonstrating why decimal matters for money

float   f1 = 0.1f + 0.2f;
double  d1 = 0.1  + 0.2;
decimal m1 = 0.1m + 0.2m;

Console.WriteLine($"float:   {f1:R}");   // 0.3000000119209290
Console.WriteLine($"double:  {d1:R}");   // 0.30000000000000004
Console.WriteLine($"decimal: {m1}");       // 0.3  ← exact!

// Always use decimal for money and financial calculations!
C# — Nullable Types Program.cs 
// Regular int cannot hold null
int a = 10;
// int b = null;   ← COMPILE ERROR

// Nullable int — can hold a number OR null
int? score    = null;    // "no value yet"
int? highScore = 98;

// Check if it has a value before using it
if (score == null)
    Console.WriteLine("Score not set yet.");

// Null-coalescing operator ?? — provide a default value
int display = score ?? 0;   // if score is null, use 0
Console.WriteLine($"Display score: {display}");   // 0

✏️ Practice Tasks

Task 1 Variable Explorer Easy

Declare one variable of each of the following types and assign it a sensible value. Then print all of them using Console.WriteLine():

  • int — your age
  • double — your height in metres (e.g., 1.75)
  • decimal — the price of a coffee (e.g., 450.00m)
  • bool — whether you are a student
  • char — the first letter of your name
  • string — your full name

Use string interpolation ($"...") for output.

Task 2 Circle Calculator Easy

Write a program that calculates the area and circumference of a circle.

  • Declare a const double Pi = 3.14159265358979
  • Declare a variable radius and assign it a value (e.g., 7.0)
  • Calculate: area = Pi * radius * radius
  • Calculate: circumference = 2 * Pi * radius
  • Print both results rounded to 2 decimal places using :F2

Tip: $"{value:F2}" formats to 2 decimal places.

Task 3 Temperature Converter Medium

Write a program that converts temperatures between Celsius and Fahrenheit.

  • Declare a double celsius variable and set it to 100.0
  • Convert to Fahrenheit: fahrenheit = celsius * 9.0 / 5.0 + 32
  • Convert to Kelvin: kelvin = celsius + 273.15
  • Print all three values clearly labelled

Bonus challenge: Also convert back from Fahrenheit to Celsius and verify that you get the original value.

⚠️ Common Mistakes

Using float or double for money
This is the most serious mistake of the lesson. double price = 0.1 + 0.2; gives 0.30000000000000004, not 0.3. Multiply that by a million transactions and you have a financial system that is wrong. Always use decimal for monetary values.
Integer division truncating: 5 / 2 = 2
When both operands are int, C# performs integer division and discards the remainder. int result = 5 / 2; gives 2, not 2.5. To get a decimal result, at least one operand must be a floating-point type: double result = 5.0 / 2; gives 2.5.
Forgetting literal suffixes: f, m, L
float x = 3.14; — compile error! The literal 3.14 is a double by default. You must write float x = 3.14f;. Similarly, decimal d = 9.99; fails — needs 9.99m. long literals over int max need the L suffix: long n = 3000000000L;.
Declaring a variable without initialization then using it
Unlike some languages, C# will not let you use an uninitialized local variable. int x; Console.WriteLine(x); is a compile error: "Use of unassigned local variable 'x'". Always assign a value before using a variable.
Confusing char quotes with string quotes
char uses single quotes: 'A'. string uses double quotes: "A". char c = "A"; is a compile error. char c = 'AB'; is also an error — a char holds exactly one character.

🎓 Instructor Notes

⚡ Teaching tips for this lesson
  • Live-code the floating-point precision demo. Run the float/double/decimal comparison example live. Seeing 0.30000000000000004 on screen has a much stronger effect than just explaining it. Then ask: "Would you trust a banking app built with double?"
  • Spend time on integer division. This is a common source of subtle bugs. Write int r = 5 / 2; and ask students what the answer is — many will say 2.5. Then show how to fix it with 5.0 / 2 or casting.
  • Introduce var but set clear expectations. Tell students that explicit types make code clearer for beginners and teammates. Use var sparingly in class examples; rely on it more as students become comfortable with types.
  • Show IntelliSense. Hover over a var declaration in Visual Studio to show what type was inferred. This builds confidence and shows the value of the IDE.
  • The data type table is reference material, not memorisation. Tell students they don't need to memorise ranges — in practice they look them up. The key things to remember: use int for integers, double for scientific math, decimal for money.
