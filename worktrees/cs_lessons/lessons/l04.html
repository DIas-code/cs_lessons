L04 — Conditional Expressions

27 February 2026 · Friday · 80 min Done

Conditional logic is the foundation of all decision-making in programs. We cover if / else if / else, comparison and logical operators, the ternary operator, and both forms of the switch statement.

🎯 Learning Objectives

  • Write if / else if / else chains to handle multiple conditions
  • Use comparison operators (==, !=, <, >, <=, >=) correctly
  • Combine conditions with logical operators (&&, ||, !)
  • Use the ternary operator for concise conditional assignment
  • Write switch statements and modern switch expressions
  • Avoid common pitfalls: = vs ==, floating-point comparison, unreachable branches

📖 Theory

1. The if Statement

The if statement executes a block of code only when a condition evaluates to true. The condition must be a bool expression — it must resolve to exactly true or false.

  • The curly braces { } define the block that belongs to the if.
  • If the block contains only one statement, the braces are optional — but always use braces anyway to avoid bugs.
  • else if adds additional conditions that are checked only when all previous conditions were false.
  • else at the end catches everything not matched by any if or else if.

C# evaluates conditions from top to bottom and stops at the first one that is true. The remaining branches are skipped entirely.

Short-circuit evaluation

&& stops evaluating as soon as it finds a false — the rest won't run. || stops as soon as it finds a true. This is important when the right side has side effects or could throw an exception.

Example: if (str != null && str.Length > 0) — if str is null, the Length check never runs, preventing a null reference exception.

4. The Ternary Operator

A compact way to write a simple if/else that produces a value. Syntax: condition ? valueIfTrue : valueIfFalse

Use it for simple, readable assignments. Do not nest ternary operators — it quickly becomes unreadable.

5. The switch Statement (Classic)

When a single variable can have many specific values, a switch is cleaner than a long chain of else if.

  • Each case must end with break (unlike C/C++, C# does not allow fall-through)
  • default is the catch-all, like else
  • Multiple case labels can share the same code block
  • Works on: int, char, string, enum

6. Switch Expression (C# 8+ — Modern)

A more concise, expression-based form that returns a value directly. It uses => instead of case/break and _ as the default wildcard. Preferred in modern C# code.

When to use what? Use if/else for range checks (>, <=) and complex conditions. Use switch when matching a variable against a list of exact values. Use the switch expression when you need to assign a value based on a match.

💻 Code Examples

C# — if / else if / else Program.cs 
Console.Write("Enter your score (0-100): ");
int score = int.Parse(Console.ReadLine()!);

if (score >= 90)
{
    Console.WriteLine("Grade: A — Excellent!");
}
else if (score >= 75)
{
    Console.WriteLine("Grade: B — Good job.");
}
else if (score >= 60)
{
    Console.WriteLine("Grade: C — Passed.");
}
else if (score >= 0)
{
    Console.WriteLine("Grade: F — Failed.");
}
else
{
    Console.WriteLine("Invalid score.");
}
C# — Logical Operators Program.cs 
int age    = 20;
bool hasID = true;

// AND: both conditions must be true
if (age >= 18 && hasID)
    Console.WriteLine("Access granted.");

// OR: at least one must be true
bool isWeekend = false;
bool isHoliday = true;
if (isWeekend || isHoliday)
    Console.WriteLine("Day off!");          // prints

// NOT: flip the boolean
bool isLoggedIn = false;
if (!isLoggedIn)
    Console.WriteLine("Please log in.");    // prints

// Short-circuit: if str is null, Length is never checked
string? str = null;
if (str != null && str.Length > 0)
    Console.WriteLine("String is not empty.");
C# — Ternary Operator Program.cs 
int age = 17;

// Without ternary:
string status1;
if (age >= 18) status1 = "Adult";
else            status1 = "Minor";

// With ternary — same thing in one line:
string status2 = age >= 18 ? "Adult" : "Minor";

Console.WriteLine(status2);   // Minor

// Ternary inside string interpolation
int n = 7;
Console.WriteLine($"{n} is {(n % 2 == 0 ? "even" : "odd")}"); // 7 is odd
C# — switch Statement (Classic) Program.cs 
Console.Write("Enter day number (1–7): ");
int day = int.Parse(Console.ReadLine()!);

switch (day)
{
    case 1:
        Console.WriteLine("Monday");
        break;
    case 2:
        Console.WriteLine("Tuesday");
        break;
    case 6:
    case 7:
        Console.WriteLine("Weekend!");    // shared by cases 6 and 7
        break;
    default:
        Console.WriteLine("Weekday");
        break;
}
C# — switch Expression (Modern, C# 8+) Program.cs 
int day = 3;

// switch expression — returns a value directly
string dayName = day switch
{
    1 => "Monday",
    2 => "Tuesday",
    3 => "Wednesday",
    4 => "Thursday",
    5 => "Friday",
    6 or 7 => "Weekend",
    _ => "Unknown"    // _ = default wildcard
};

Console.WriteLine(dayName);   // Wednesday

// Practical example: traffic light
string light = "red";
string action = light switch
{
    "red"    => "Stop",
    "yellow" => "Slow down",
    "green"  => "Go",
    _         => "Invalid signal"
};
Console.WriteLine(action);   // Stop

⚠️ Common Mistakes

Assignment = instead of comparison ==
if (x = 5) is a compile error in C# (unlike C/C++ where it silently works). C# requires the condition to be a bool, and x = 5 is an assignment (not bool). This is one case where the compiler protects you — but still be careful.
Comparing floating-point numbers with ==
if (0.1 + 0.2 == 0.3) evaluates to false due to floating-point precision. For doubles, compare with a tolerance: Math.Abs(a - b) < 0.0001. For decimal, == is fine.
Forgetting break in a switch statement
Every case block must end with break, return, or throw. Forgetting break is a compile error in C# (C# does not allow fall-through). This is actually a safety feature.
Overlapping else if conditions — order matters
If you write the broadest condition first, the more specific ones will never be reached. Example: if (score >= 60) before if (score >= 90) — a score of 95 will match the first branch and "Excellent" will never print. Always write from most specific to most general (or use proper non-overlapping ranges).

🎓 Instructor Notes

⚡ Teaching tips
  • Draw flowcharts for the first examples. Before writing code, draw a diamond decision box for the grade checker. Students who struggle with code often understand flowcharts immediately — this bridges the gap.
  • Live-demo the order-of-conditions bug. Write the grade checker with the conditions in wrong order (broad first) and ask students why everything prints "C". Then fix it together.
  • Show both switch styles. Teach the classic switch for recognition (they'll see it everywhere), then show the switch expression and note that it is preferred in new code.
