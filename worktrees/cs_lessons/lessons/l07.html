Part 1 — C# Basics

L07 — Methods

20 March 2026 · Friday · 80 min Done

Methods are named, reusable blocks of code. They are the most important tool for structuring a program — breaking it into smaller, focused, testable pieces. We cover declaration, parameters, return values, overloading, and scope.

🎯 Learning Objectives

  • Declare and call methods with and without parameters
  • Distinguish between void methods and methods that return a value
  • Use return to send a value back to the caller
  • Define methods with default parameter values and call them with named arguments
  • Overload methods (same name, different parameter lists)
  • Understand variable scope — local variables vs. method parameters
  • Pass arrays to methods and understand reference vs. value semantics

📖 Theory

1. Why Methods?

Without methods, code is a giant sequential block that is impossible to read, debug, or reuse. Methods let you:

  • Name a piece of logic — the name documents intent
  • Reuse it — call it many times without copying code
  • Test it in isolation — a method with clear inputs and outputs is easy to verify
  • Hide details — the caller doesn't need to know how it works, only what it does
Single Responsibility Principle (preview). A good method does exactly one thing and does it well. If you find yourself writing a method called ReadInputAndCalculateAndPrint(), split it into three separate methods.

2. Method Declaration Syntax

The full syntax of a method declaration:

Syntax
// [access] [modifiers] returnType MethodName([parameters])
// {
//     body
//     return value;   ← required if returnType is not void
// }

static int Add(int a, int b)
{
    return a + b;
}
  • static — for now, all our methods are static (called on the class, not an object). This changes when we learn OOP.
  • Return type — the type of value the method sends back. Use void if it returns nothing.
  • Method name — PascalCase by convention: CalculateArea, PrintResult, GetMaxValue
  • Parameters — comma-separated list of type name pairs, or empty ()

3. Parameters vs. Arguments

These two terms are often confused:

  • Parameter — the variable declared in the method signature: int a, int b
  • Argument — the actual value passed when the method is called: Add(3, 7)

4. The return Statement

  • Returns a value from the method to the caller. The value must match the declared return type.
  • Immediately exits the method — any code after return in the same path is unreachable.
  • A void method can use bare return; to exit early (optional at the end).
  • A method can have multiple return statements (e.g., one per branch in an if/else).

5. Default Parameter Values

Parameters can have default values. If the caller does not provide an argument for that parameter, the default is used. Default parameters must come after required parameters.

Example: static void Greet(string name, string greeting = "Hello")

Call as Greet("Alice") → uses default → "Hello, Alice!"

Call as Greet("Alice", "Hi") → overrides default → "Hi, Alice!"

6. Named Arguments

When calling a method, you can name the arguments explicitly: Greet(greeting: "Hey", name: "Bob") This allows you to pass them in any order and makes the call self-documenting for methods with many parameters.

7. Method Overloading

Multiple methods can share the same name as long as they have different parameter lists (different number of parameters, or different types). The compiler picks the correct version based on the arguments you pass. The return type alone does NOT distinguish overloads.

Classic example: Console.WriteLine() is overloaded dozens of times — for int, double, string, bool, etc.

8. Variable Scope

A variable declared inside a method is local to that method. It does not exist outside of it — each call to the method gets its own fresh copy of local variables.

  • Parameters are local variables initialised by the caller's arguments
  • Two different methods can have variables with the same name — they are completely independent
  • Variables declared inside a block (if, for) are scoped to that block only

9. Passing Arrays to Methods

Arrays are reference types. When you pass an array to a method, you are passing a reference to the same data in memory — not a copy. This means modifications made to the array inside the method will affect the original array in the caller.

This is a key difference from value types like int, which are copied on pass.

10. Expression-Bodied Methods (C# 6+)

For methods that consist of a single expression, you can use the => arrow syntax to write them on one line. This is purely stylistic — identical in behaviour.

static int Square(int n) => n * n;

💻 Code Examples

C# — void Method and Return Method Program.cs 
// void method — does something, returns nothing
static void PrintLine(string text)
{
    Console.WriteLine("→ " + text);
}

// method with return value
static double CircleArea(double radius)
{
    return Math.PI * radius * radius;
}

// expression-bodied (same as above, one line)
static int Square(int n) => n * n;

// --- calling them ---
PrintLine("Hello!");                             // → Hello!

double area = CircleArea(5.0);
Console.WriteLine($"Area: {area:F2}");            // Area: 78.54

Console.WriteLine(Square(7));                    // 49
C# — Multiple return statements + early exit Program.cs 
// Method with multiple return paths
static string GetGrade(int score)
{
    if (score < 0 || score > 100) return "Invalid";
    if (score >= 90) return "A";
    if (score >= 75) return "B";
    if (score >= 60) return "C";
    return "F";
}

Console.WriteLine(GetGrade(93));   // A
Console.WriteLine(GetGrade(55));   // F
Console.WriteLine(GetGrade(-5));   // Invalid
C# — Default Parameters and Named Arguments Program.cs 
static void Greet(string name, string greeting = "Hello", bool shout = false)
{
    string msg = $"{greeting}, {name}!";
    Console.WriteLine(shout ? msg.ToUpper() : msg);
}

Greet("Alice");                              // Hello, Alice!
Greet("Bob", "Hi");                         // Hi, Bob!
Greet("Carol", "Hey", true);               // HEY, CAROL!

// Named arguments — order doesn't matter
Greet(shout: true, name: "Dave", greeting: "Yo");  // YO, DAVE!
C# — Method Overloading Program.cs 
// Three overloads of Print — same name, different parameter types
static void Print(int value)    => Console.WriteLine($"int:    {value}");
static void Print(double value) => Console.WriteLine($"double: {value}");
static void Print(string value) => Console.WriteLine($"string: {value}");
static void Print(int a, int b)  => Console.WriteLine($"sum:    {a + b}");

// Compiler chooses the right overload automatically
Print(42);           // int:    42
Print(3.14);         // double: 3.14
Print("hello");      // string: hello
Print(3, 7);          // sum:    10
C# — Passing Arrays (Reference Semantics) Program.cs 
// Arrays are passed by reference — changes inside affect the original
static void DoubleAll(int[] arr)
{
    for (int i = 0; i < arr.Length; i++)
        arr[i] *= 2;
}

static int Sum(int[] arr)
{
    int total = 0;
    foreach (int n in arr) total += n;
    return total;
}

int[] nums = { 1, 2, 3, 4, 5 };
Console.WriteLine($"Sum before: {Sum(nums)}");   // 15

DoubleAll(nums);   // modifies the ORIGINAL array
Console.WriteLine($"Sum after:  {Sum(nums)}");   // 30
Console.WriteLine(string.Join(", ", nums));      // 2, 4, 6, 8, 10
C# — Well-structured Program using Methods Program.cs 
// Top-level: read input, calculate, display
int[] data = ReadArray(5);
PrintStats(data);

// ---- method definitions ----

static int[] ReadArray(int size)
{
    int[] arr = new int[size];
    for (int i = 0; i < size; i++)
    {
        Console.Write($"Enter number {i + 1}: ");
        arr[i] = int.Parse(Console.ReadLine()!);
    }
    return arr;
}

static void PrintStats(int[] arr)
{
    int sum = 0, min = arr[0], max = arr[0];
    foreach (int n in arr)
    {
        sum += n;
        if (n < min) min = n;
        if (n > max) max = n;
    }
    Console.WriteLine($"Sum={sum}  Avg={(double)sum/arr.Length:F1}  Min={min}  Max={max}");
}

⚠️ Common Mistakes

Calling a method without using its return value
GetGrade(85); — this calls the method but throws away the result. The method runs but nothing happens. You need to either store the result (string g = GetGrade(85);) or use it directly (Console.WriteLine(GetGrade(85));).
Thinking int arguments are shared with the caller
Value types (int, double, bool) are copied when passed to a method. Changing the parameter inside the method has no effect on the original variable. Only arrays and objects (reference types) share the underlying data.
Method declared after the top-level call but not found
In a top-level statements file, all static local methods must be declared after the top-level code, not inside it. If you accidentally put a method definition in the middle of executable code, you get a compile error. Keep all method definitions at the bottom of the file, below all executable statements.
Overloads with the same signature — compile error
static int Add(int a, int b) and static double Add(int a, int b) will not compile — the parameter list is identical. Return type does not differentiate overloads. The compiler cannot determine which one to call based only on the arguments.

🎓 Instructor Notes

⚡ Teaching tips
  • Refactor live. Start with a 30-line top-level program (read array, sort, print stats — all inline). Then say "Let's clean this up" and extract it into methods step by step. Students see the before/after and understand why methods exist.
  • Demo array reference semantics clearly. Write void AddOne(int n) { n++; } and void AddOneToAll(int[] arr) { ... }, run both, show that the first changes nothing outside, the second does. This is a critical conceptual moment.
  • Give Task 4 today — it is the most substantial Part 1 assignment. Walk through the requirements. Students who are struggling with Tasks 1–3 need to start immediately.
  • Introduce the Single Responsibility idea early. Students who grasp it now write dramatically better code in the OOP section.
