Part 1 — C# Basics

L06 — Arrays

13 March 2026 · Friday · 80 min Done

An array is the most fundamental data structure — a fixed-size, ordered collection of elements of the same type. We learn to declare, initialize, iterate, and manipulate arrays, including 2D arrays and the most useful methods from the Array class.

🎯 Learning Objectives

  • Declare and initialize 1D arrays using both allocation and initializer syntax
  • Access and modify array elements using zero-based indexing
  • Iterate arrays with for (index-based) and foreach
  • Use key Array class methods: Sort, Reverse, IndexOf, Copy
  • Declare and work with 2D (rectangular) arrays
  • Understand why arrays have a fixed length and when to prefer other collections

📖 Theory

1. What Is an Array?

An array is a fixed-size, contiguous block of memory that holds multiple elements of the same type, accessible by an integer index.

Key characteristics:

  • Fixed size — once created, the length cannot change
  • Zero-based — the first element is at index 0, the last at Length - 1
  • Same type — all elements must be the same declared type
  • Reference type — the array variable holds a reference to the data on the heap

2. Declaring and Initializing Arrays

There are several ways to create an array:

Method A — Allocate with new, fill later

int[] numbers = new int[5]; Creates an array of 5 integers. All elements are initialised to their default value: 0 for numeric types, false for bool, null for reference types like string.

Method B — Array initializer (size inferred)

int[] numbers = { 10, 20, 30, 40, 50 }; The size is inferred from the number of values. Clean and concise — preferred when you know the values upfront.

Method C — new with explicit initializer

int[] numbers = new int[] { 10, 20, 30 }; Equivalent to Method B, but verbose. Sometimes needed in expressions.

3. Accessing Elements

Elements are accessed using square brackets with a zero-based index.

  • arr[0] — first element
  • arr[arr.Length - 1] — last element
  • arr[^1] — last element (C# 8+ index-from-end syntax)
  • arr[^2] — second from last

Accessing an index outside 0 to Length-1 throws an IndexOutOfRangeException at runtime.

4. The Length Property

Every array has a Length property that returns the total number of elements. Always use arr.Length instead of hardcoding the size — your loop will still work correctly if the array size changes.

5. Iterating Arrays

Two main approaches:

  • for loop with index — use when you need the index, need to modify elements, or need to iterate in non-default order (backwards, skip every other, etc.)
  • foreach loop — use when you only need the value and don't need the index. Cleaner code, no chance of off-by-one errors. Cannot modify elements.

6. The Array Class — Useful Static Methods

MethodDescription
Array.Sort(arr)Sorts elements in ascending order (in-place — modifies the original array)
Array.Reverse(arr)Reverses the order of elements (in-place)
Array.IndexOf(arr, value)Returns the index of the first occurrence of value. Returns -1 if not found.
Array.Copy(src, dest, count)Copies count elements from src into dest
Array.Fill(arr, value)Sets all elements to the specified value (.NET 5+)
Array.Clear(arr)Resets all elements to their default value (0, false, null)
Sort and Reverse modify the original array. If you need to preserve the original order, copy the array first with Array.Copy() or arr.Clone() before sorting.

7. 2D Arrays (Rectangular)

A 2D array stores data in rows and columns — like a table or matrix. Declared with a comma inside the brackets: int[,]

  • int[,] matrix = new int[3, 4]; — 3 rows, 4 columns
  • Access: matrix[row, col]
  • matrix.GetLength(0) — number of rows
  • matrix.GetLength(1) — number of columns
  • matrix.Length — total elements (rows × cols)

8. Arrays vs. Other Collections

Arrays are fast and simple, but their fixed size is a limitation. When you need a dynamically sized collection, use List<T> instead. We will cover collections in the OOP section.

💻 Code Examples

C# — Array Declaration and Access Program.cs 
// Method A: allocate first, then assign
int[] scores = new int[5];
scores[0] = 85;
scores[1] = 92;
scores[2] = 78;
scores[3] = 96;
scores[4] = 61;

// Method B: initializer — cleaner
string[] names = { "Alice", "Bob", "Carol", "Dave" };

Console.WriteLine($"Length: {names.Length}");         // 4
Console.WriteLine($"First: {names[0]}");              // Alice
Console.WriteLine($"Last:  {names[names.Length - 1]}"); // Dave
Console.WriteLine($"Last:  {names[^1]}");              // Dave (C# 8+ syntax)
C# — Iterating: for vs foreach Program.cs 
int[] nums = { 10, 20, 30, 40, 50 };

// for — when you need the index
for (int i = 0; i < nums.Length; i++)
{
    Console.WriteLine($"[{i}] = {nums[i]}");
    nums[i] *= 2;   // can modify elements
}

// foreach — clean when you just need values
foreach (int n in nums)
    Console.Write(n + " ");   // 20 40 60 80 100
C# — Array Class Methods Program.cs 
int[] arr = { 5, 2, 8, 1, 9, 3 };

Array.Sort(arr);
Console.WriteLine(string.Join(", ", arr));   // 1, 2, 3, 5, 8, 9

Array.Reverse(arr);
Console.WriteLine(string.Join(", ", arr));   // 9, 8, 5, 3, 2, 1

// Search: IndexOf returns -1 if not found
int idx = Array.IndexOf(arr, 5);
Console.WriteLine($"5 is at index {idx}");   // 5 is at index 2

// Find min and max manually
int min = arr[0], max = arr[0];
foreach (int n in arr)
{
    if (n < min) min = n;
    if (n > max) max = n;
}
Console.WriteLine($"Min={min}, Max={max}");
C# — 2D Array (Matrix) Program.cs 
// 3x3 matrix with initializer
int[,] matrix = {
    { 1, 2, 3 },
    { 4, 5, 6 },
    { 7, 8, 9 }
};

int rows = matrix.GetLength(0);   // 3
int cols = matrix.GetLength(1);   // 3

// Print matrix using nested loops
for (int r = 0; r < rows; r++)
{
    for (int c = 0; c < cols; c++)
        Console.Write($"{matrix[r, c],4}");
    Console.WriteLine();
}
/*
   1   2   3
   4   5   6
   7   8   9
*/

// Sum of diagonal (top-left to bottom-right)
int diagSum = 0;
for (int i = 0; i < rows; i++)
    diagSum += matrix[i, i];   // 1 + 5 + 9 = 15
Console.WriteLine($"Diagonal sum: {diagSum}");
C# — Reading an Array from the User Program.cs 
Console.Write("How many numbers? ");
int n = int.Parse(Console.ReadLine()!);

int[] arr = new int[n];

for (int i = 0; i < n; i++)
{
    Console.Write($"Enter number [{i + 1}]: ");
    arr[i] = int.Parse(Console.ReadLine()!);
}

Array.Sort(arr);
Console.WriteLine("Sorted: " + string.Join(", ", arr));

int sum = 0;
foreach (int x in arr) sum += x;
Console.WriteLine($"Sum: {sum}, Avg: {(double)sum / n:F2}");

⚠️ Common Mistakes

IndexOutOfRangeException — accessing beyond array bounds
The most frequent array runtime error. Accessing arr[arr.Length] instead of arr[arr.Length - 1] throws immediately. Always remember: last valid index is Length - 1. Use arr.Length as the loop condition, not <= arr.Length.
Hardcoding the array size in a loop
for (int i = 0; i < 5; i++) — if you later change the array from 5 to 7 elements, the loop won't cover all elements and no error will appear. Always write i < arr.Length so the loop adapts automatically.
Trying to resize an array after creation
Arrays in C# have a fixed size. arr.Length = 10; is a compile error — Length is read-only. If you need to add/remove elements dynamically, use List<int> instead.
Sorting the original when you wanted a copy
Array.Sort() modifies the array in place. If you need the original order preserved, copy first: int[] sorted = (int[])arr.Clone(); then sort sorted.

🎓 Instructor Notes

⚡ Teaching tips
  • Draw memory boxes for the first example. Sketch 5 boxes labelled 0–4 on the board, then fill them as you write the code. The visual "array = boxes in a row" model sticks well and prevents index confusion.
  • Force the IndexOutOfRange live. Write arr[arr.Length] intentionally and run it. Reading the red exception message in the console teaches students to decode runtime errors themselves.
  • Introduce string.Join() early. Printing an array with string.Join(", ", arr) is dramatically cleaner than a manual loop. Students will use it constantly — introduce it here.
  • 2D arrays — keep it visual. Draw the grid on the board with row/col indices labeled. Students who struggle with nested loops immediately understand once they see the grid with coordinates.
