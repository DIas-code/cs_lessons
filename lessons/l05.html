L05 — Loops

06 March 2026 · Friday · 80 min Done

Loops allow a program to repeat a block of code. We cover all four loop types in C# — for, while, do-while, and foreach — along with break, continue, and nested loops.

🎯 Learning Objectives

  • Choose the correct loop type for a given scenario
  • Write for loops with custom init, condition, and step expressions
  • Write while and do-while loops for condition-driven repetition
  • Use foreach to iterate collections without managing an index
  • Control loop flow with break and continue
  • Write and reason about nested loops
  • Identify and fix infinite loops

📖 Theory

1. The for Loop

Use a for loop when you know exactly how many times the loop should run, or when you need an index variable to track the position.

Syntax: for (initializer; condition; iterator) { body }

  • Initializer — runs once before the loop starts (e.g., int i = 0)
  • Condition — checked before each iteration; loop runs while true
  • Iterator — runs after each iteration (e.g., i++, i += 2, i--)

The loop variable i is scoped to the loop — you cannot use it after the closing brace.

A for loop can count backwards: for (int i = 10; i >= 0; i--)

It can count by any step: for (int i = 0; i <= 100; i += 5)

2. The while Loop

Use a while loop when you don't know in advance how many iterations are needed — you repeat as long as a condition remains true. The condition is checked before each iteration — if it is false from the start, the body never executes.

Classic use: read user input until valid input is given, or process a file until end-of-file.

3. The do-while Loop

Like while, but the condition is checked after each iteration. This guarantees the body executes at least once, regardless of the condition.

Use case: show a menu to the user — always show it at least once, then repeat if they choose "again".

4. The foreach Loop

Designed for iterating over collections (arrays, lists, etc.) without needing an index. Cleaner and less error-prone than a for loop when you only need the value.

  • Cannot modify the collection while iterating
  • The iteration variable is readonly — you cannot assign to it
  • Use a for loop instead when you need the index or need to modify elements

5. break and continue

KeywordEffectUse when...
break Immediately exits the loop — execution continues after the closing brace You found what you were looking for, or an exit condition is met mid-loop
continue Skips the rest of the current iteration and jumps to the next one You want to skip certain values (e.g., skip even numbers, skip empty strings)

In a nested loop, break and continue only affect the innermost loop.

6. Nested Loops

A loop inside another loop. The inner loop runs its full cycle for every single iteration of the outer loop. Total iterations = outer_count × inner_count.

Classic use: printing 2D grids, multiplication tables, working with 2D arrays.

7. Infinite Loops

A loop whose condition never becomes false. This causes the program to hang. Always ensure:

  • The loop variable is actually modified inside the loop
  • The condition can eventually become false
  • A break statement provides an exit path if using while (true)

while (true) { ... break; } is a valid and common pattern — used for "run forever until explicitly broken".

Which loop to use — quick guide:
  • Know the count in advance → for
  • Repeat until a condition changes, check before → while
  • Must run at least once → do-while
  • Iterating a collection, no index needed → foreach

💻 Code Examples

C# — for Loop Variations Program.cs 
// Count up
for (int i = 1; i <= 5; i++)
    Console.Write(i + " ");       // 1 2 3 4 5
Console.WriteLine();

// Count down
for (int i = 5; i >= 1; i--)
    Console.Write(i + " ");       // 5 4 3 2 1
Console.WriteLine();

// Step by 2 — only even numbers
for (int i = 0; i <= 10; i += 2)
    Console.Write(i + " ");       // 0 2 4 6 8 10
Console.WriteLine();

// Sum from 1 to 100
int sum = 0;
for (int i = 1; i <= 100; i++)
    sum += i;
Console.WriteLine($"Sum 1–100 = {sum}");   // 5050
C# — while Loop Program.cs 
// Keep asking until valid positive number is entered
int number = 0;
while (number <= 0)
{
    Console.Write("Enter a positive number: ");
    int.TryParse(Console.ReadLine(), out number);
}
Console.WriteLine($"You entered: {number}");

// Countdown with while
int countdown = 5;
while (countdown > 0)
{
    Console.WriteLine(countdown--);   // prints then decrements
}
Console.WriteLine("Go!");
C# — do-while Loop Program.cs 
// Menu that always shows at least once
string? choice;
do
{
    Console.WriteLine("--- MENU ---");
    Console.WriteLine("1. Say hello");
    Console.WriteLine("2. Exit");
    Console.Write("Your choice: ");
    choice = Console.ReadLine();

    if (choice == "1")
        Console.WriteLine("Hello!");

} while (choice != "2");   // repeat until user chooses Exit

Console.WriteLine("Goodbye!");
C# — break and continue Program.cs 
// break — exit loop early when target found
for (int i = 1; i <= 10; i++)
{
    if (i == 5)
    {
        Console.WriteLine("Found 5, stopping.");
        break;
    }
    Console.Write(i + " ");   // prints: 1 2 3 4
}
Console.WriteLine();

// continue — skip even numbers
for (int i = 1; i <= 10; i++)
{
    if (i % 2 == 0) continue;   // skip even
    Console.Write(i + " ");       // prints: 1 3 5 7 9
}
C# — Nested Loops: Multiplication Table Program.cs 
// Print multiplication table 1–5
for (int row = 1; row <= 5; row++)
{
    for (int col = 1; col <= 5; col++)
    {
        Console.Write($"{row * col,4}");  // ,4 = pad to 4 chars wide
    }
    Console.WriteLine();   // new line after each row
}
/*
Output:
   1   2   3   4   5
   2   4   6   8  10
   3   6   9  12  15
   4   8  12  16  20
   5  10  15  20  25
*/
C# — foreach Loop Program.cs 
string[] cities = { "Almaty", "Astana", "Shymkent", "Karaganda" };

// foreach — clean, no index management
foreach (string city in cities)
{
    Console.WriteLine(city);
}

// Sum an array of numbers with foreach
int[] scores = { 85, 92, 78, 96, 61 };
int total = 0;
foreach (int s in scores)
    total += s;
Console.WriteLine($"Total: {total}, Average: {(double)total / scores.Length:F1}");

⚠️ Common Mistakes

Off-by-one error (fencepost error)
for (int i = 1; i < 10; i++) runs 9 times (1–9). for (int i = 1; i <= 10; i++) runs 10 times (1–10). Always double-check whether the boundary is inclusive (<=) or exclusive (<).
Forgetting to modify the loop variable — infinite loop
int i = 0; while (i < 10) { Console.WriteLine(i); } — i never changes, the loop runs forever. Always make sure the variable driving the condition is updated inside the loop body.
Semicolon after the for header
for (int i = 0; i < 5; i++); — the semicolon creates an empty loop body. The loop runs 5 times doing nothing, then the block below runs exactly once. This is a sneaky bug because the code compiles without error.
Using break expecting it to exit an outer loop
In nested loops, break only exits the innermost loop. To exit multiple levels, you need a flag variable or restructure the logic. Example: bool found = false; set inside inner loop, checked in outer loop condition.

🎓 Instructor Notes

⚡ Teaching tips
  • FizzBuzz is the live-coding warmup. Most students will attempt it with too many if conditions. Guide them to discover that %15 == 0 OR checking both simultaneously is the clean solution.
  • Demonstrate an infinite loop live — carefully. Write one intentionally, show the program hanging, then Ctrl+C to kill it. Explain what happened. Students need to see this to recognize it in their own code.
  • Nested loops: trace by hand first. Before running the multiplication table, trace through the first two rows on the board — row=1 col=1, row=1 col=2... This mental model is crucial.
  • Give Task 3 at the end. It is intentionally short (2 pts) — it is a warm-up assignment to establish the submission workflow before the heavier tasks come in later lessons.
