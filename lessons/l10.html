1. What is Object-Oriented Programming?

So far we have written procedural code — a sequence of instructions, loops, and methods operating on primitive data. As programs grow, this becomes hard to manage: data and the code that operates on it are spread everywhere.

Object-Oriented Programming (OOP) solves this by organising code around objects — bundles that hold both data (fields/properties) and behaviour (methods) together.

The four pillars of OOP
We will cover all four across L10 and L11:
  1. Encapsulation — hiding internal details, exposing a clean interface (today)
  2. Abstraction — modelling only what is relevant (today)
  3. Inheritance — reusing and extending existing classes (L11)
  4. Polymorphism — one interface, many forms (L11)

Procedural vs OOP — a side-by-side

Procedural approachC#
// Data lives in separate variables
string studentName = "Aisha";
int    studentAge  = 20;
double gpa         = 3.8;

// Behaviour is a disconnected method
void PrintStudent(string name, int age, double gpa)
{
    Console.WriteLine($"{name}, age {age}, GPA {gpa}");
}
OOP approachC#
// Data and behaviour live together in a class
class Student
{
    public string Name { get; set; }
    public int    Age  { get; set; }
    public double GPA  { get; set; }

    public void Print()
    {
        Console.WriteLine($"{Name}, age {Age}, GPA {GPA}");
    }
}

Student s = new Student { Name = "Aisha", Age = 20, GPA = 3.8 };
s.Print();

2. Classes and Objects

Class vs Object (Instance)

A class is a blueprint — it describes what shape an object will have. An object is a concrete instance built from that blueprint. You can create many objects from one class, each with its own data.

Analogy: A class is like an architectural drawing for a house. An object is an actual house built from that drawing. You can build many houses (objects) from one drawing (class).
Creating and using objectsC#
class Car
{
    public string Brand { get; set; }
    public int    Year  { get; set; }
    public int    Speed { get; set; }

    public void Accelerate(int amount) => Speed += amount;
    public void Brake(int amount)     => Speed = Math.Max(0, Speed - amount);

    public void PrintStatus()
        => Console.WriteLine($"{Brand} ({Year}) — {Speed} km/h");
}

// Two separate objects — independent data
Car car1 = new Car { Brand = "Toyota", Year = 2024 };
Car car2 = new Car { Brand = "BMW",    Year = 2025 };

car1.Accelerate(60);
car2.Accelerate(100);

car1.PrintStatus();  // Toyota (2024) — 60 km/h
car2.PrintStatus();  // BMW (2025) — 100 km/h

Where to place classes

In .NET 10 top-level programs, define classes after the top-level statements:

Program.cs — class after top-level codeC#
// Top-level statements come first
Car myCar = new Car { Brand = "Lada", Year = 1999 };
myCar.Accelerate(40);
myCar.PrintStatus();

// Class definitions come after
class Car
{
    public string Brand { get; set; }
    public int    Year  { get; set; }
    public int    Speed { get; set; }

    public void Accelerate(int amount) => Speed += amount;
    public void PrintStatus()
        => Console.WriteLine($"{Brand} ({Year}) — {Speed} km/h");
}
Best practice: For larger projects, put each class in its own file (e.g., Car.cs, Student.cs). For today's exercises, putting everything in Program.cs is fine.

3. Fields

A field is a variable declared directly inside a class (not inside a method). Fields store the state of an object.

Fields — declaration and defaultsC#
class BankAccount
{
    // Fields — per-instance data
    public string Owner;
    public decimal Balance;       // default: 0
    public bool    IsActive;      // default: false
    public string  AccountNumber; // default: null

    // Field with explicit default
    public decimal InterestRate = 0.05m;
}
Public fields are usually bad. Anyone can write account.Balance = -999999; and nothing will stop them. Use properties with controlled access instead (next section).

4. Access Modifiers

Access modifiers control who can see and modify members of a class.

Modifier Visible to Typical use
public Everyone Properties, methods that form the public API
private Only this class Fields, helper methods (default if omitted)
protected This class + subclasses Data shared with derived classes (L11)
internal Same assembly/project Library classes not meant for external use
Rule of thumb: Make everything private by default. Only make something public when external code genuinely needs it. This is the core of encapsulation.

5. Properties

A property is a member that looks like a field from the outside but uses get/set accessors — so the class can validate or transform values.

Full property with backing field

Manual property with validationC#
class Person
{
    private int _age; // backing field — private, convention: _camelCase

    public int Age
    {
        get { return _age; }
        set
        {
            if (value < 0 || value > 150)
                throw new ArgumentOutOfRangeException("Age must be 0–150");
            _age = value;
        }
    }
}

Person p = new Person();
p.Age = 25;   // OK — calls set accessor
p.Age = -5;   // throws ArgumentOutOfRangeException
int a = p.Age; // OK — calls get accessor

Auto-property (most common)

When you don't need validation, C# can generate the backing field automatically. This is called an auto-implemented property.

Auto-propertiesC#
class Product
{
    public string  Name     { get; set; }   // read + write
    public decimal Price    { get; set; }
    public int     Stock    { get; set; }
    public string  Category { get; }        // read-only (set only in constructor)

    // Computed property — no setter, derived from other data
    public bool IsAvailable => Stock > 0;
    public decimal PriceWithTax => Price * 1.12m;
}

Product prod = new Product { Name = "Laptop", Price = 850m, Stock = 5 };
Console.WriteLine(prod.IsAvailable);   // True
Console.WriteLine(prod.PriceWithTax);  // 952.00

Property accessibility levels

init-only and private setC#
class Employee
{
    // Can only be set at object creation (object initializer or constructor)
    public string Id   { get; init; }

    // Can be set freely inside the class, read-only outside
    public int LoginCount { get; private set; }

    public void RecordLogin() => LoginCount++;
}

Employee emp = new Employee { Id = "EMP001" };
emp.RecordLogin();
Console.WriteLine(emp.LoginCount); // 1
// emp.Id = "EMP002";         // ERROR — init-only
// emp.LoginCount = 99;       // ERROR — private set

6. Constructors

A constructor is a special method that runs when an object is created with new. Its job is to put the object into a valid initial state.

Default constructor

If you define no constructor, C# provides a parameterless one that sets all fields to their defaults.

Custom constructors

Constructor basicsC#
class Student
{
    public string Name    { get; set; }
    public int    Age     { get; set; }
    public double GPA     { get; set; }
    public string Faculty { get; set; }

    // Parameterless constructor
    public Student()
    {
        Faculty = "Undeclared";
        GPA     = 0.0;
    }

    // Parameterised constructor
    public Student(string name, int age, string faculty)
    {
        Name    = name;
        Age     = age;
        Faculty = faculty;
        GPA     = 0.0;
    }
}

// Three ways to create a Student
Student s1 = new Student();                              // parameterless
Student s2 = new Student("Dias", 21, "IT");              // parameterised
Student s3 = new Student { Name = "Aisha", Age = 20 };   // object initializer

Constructor chaining with this()

One constructor can call another to avoid code duplication:

Constructor chainingC#
class Rectangle
{
    public double Width  { get; }
    public double Height { get; }

    // Main constructor
    public Rectangle(double width, double height)
    {
        Width  = width;
        Height = height;
    }

    // Square shortcut — delegates to main constructor
    public Rectangle(double side) : this(side, side) { }

    public double Area()      => Width * Height;
    public double Perimeter() => 2 * (Width + Height);
}

Rectangle r1 = new Rectangle(4, 6);
Rectangle r2 = new Rectangle(5);    // 5×5 square

Console.WriteLine(r1.Area());      // 24
Console.WriteLine(r2.Perimeter()); // 20

Static constructor

A static constructor runs once the first time the class is used. Used to initialise static data:

Static constructorC#
class AppConfig
{
    public static string Version { get; }
    public static string AppName { get; }

    // Runs once — no parameters allowed
    static AppConfig()
    {
        Version = "1.0.0";
        AppName = "My App";
    }
}

Console.WriteLine(AppConfig.Version); // 1.0.0

7. The this Keyword

Inside a class, this refers to the current instance. Most common uses:

Using this to disambiguateC#
class Circle
{
    public double Radius { get; set; }

    // Parameter "radius" shadows the property — use this.Radius to mean the field
    public Circle(double radius)
    {
        this.Radius = radius; // "this.Radius" = property; "radius" = parameter
    }

    public double Area()         => Math.PI * Radius * Radius;
    public double Circumference() => 2 * Math.PI * Radius;

    // Fluent interface — return this to allow chaining
    public Circle Scale(double factor) { Radius *= factor; return this; }
}

Circle c = new Circle(5);
Console.WriteLine(c.Scale(2).Area());  // 314.16... (r=10)

8. Static Members

A static member belongs to the class itself, not to any particular instance. All instances share the same static data.

Static field and methodC#
class Counter
{
    private static int _total = 0;  // shared across all instances
    public int Id { get; }

    public Counter()
    {
        _total++;
        Id = _total;
    }

    public static int GetTotal() => _total;
}

Counter a = new Counter(); // Id=1, total=1
Counter b = new Counter(); // Id=2, total=2
Counter c = new Counter(); // Id=3, total=3

Console.WriteLine(Counter.GetTotal()); // 3
Instance memberStatic member
Accessed via object: obj.Method()Accessed via class: Class.Method()
Each object has its own copy of dataOne shared copy for all objects
Can use thisCannot use this
Example: car.SpeedExample: Math.Sqrt(), Console.WriteLine()

9. Value Types vs Reference Types (Revisited)

We saw this briefly in L02. Now it matters practically because classes are reference types.

Reference type behaviourC#
Student s1 = new Student("Aisha", 20, "IT");
Student s2 = s1;  // s2 points to the SAME object, not a copy!

s2.Name = "Bekzod";

Console.WriteLine(s1.Name); // Bekzod  ← changed through s2
Console.WriteLine(s2.Name); // Bekzod
Trap: Assigning one object variable to another does NOT copy the object — both variables point to the same object in memory. To copy, you need to create a new instance and copy properties manually, or use a pattern like MemberwiseClone().
Null referenceC#
Student s = null;     // valid — s holds no object
s.Name = "Test";      // NullReferenceException at runtime!

// Always check before using:
if (s != null)
    Console.WriteLine(s.Name);

// Modern null-conditional operator:
Console.WriteLine(s?.Name ?? "(no student)");

10. Complete Example — BankAccount

Putting everything together: fields, properties, constructors, methods.

BankAccount classC#
// Program.cs
BankAccount acc1 = new BankAccount("Dias", 50000m);
BankAccount acc2 = new BankAccount("Aisha");

acc1.Deposit(10000m);
acc1.Transfer(acc2, 5000m);
acc1.PrintStatement();
acc2.PrintStatement();

Console.WriteLine($"Total accounts: {BankAccount.GetAccountCount()}");

// ─────────── class definition ───────────
class BankAccount
{
    private static int  _accountCount = 0;
    private decimal     _balance;

    public string  Owner         { get; }
    public string  AccountNumber { get; }

    public decimal Balance
    {
        get => _balance;
        private set
        {
            if (value < 0)
                throw new InvalidOperationException("Balance cannot go negative");
            _balance = value;
        }
    }

    public BankAccount(string owner, decimal initialDeposit = 0)
    {
        Owner         = owner;
        Balance       = initialDeposit;
        _accountCount++;
        AccountNumber = $"ACC{_accountCount:D4}";
    }

    public void Deposit(decimal amount)
    {
        if (amount <= 0) throw new ArgumentException("Deposit must be positive");
        Balance += amount;
    }

    public void Withdraw(decimal amount)
    {
        if (amount <= 0) throw new ArgumentException("Amount must be positive");
        Balance -= amount; // Balance setter guards against negative
    }

    public void Transfer(BankAccount target, decimal amount)
    {
        Withdraw(amount);
        target.Deposit(amount);
    }

    public void PrintStatement()
        => Console.WriteLine($"[{AccountNumber}] {Owner}: {Balance:C}");

    public static int GetAccountCount() => _accountCount;
}

Expected output:

[ACC0001] Dias: ₸55,000.00
[ACC0002] Aisha: ₸5,000.00
Total accounts: 2

Instructor Notes

  • Live-code the Student class from scratch — start with public fields, then show the problem (invalid age), then convert to a property with validation.
  • Emphasise: a class is a contract. Once code outside the class calls methods, the internal implementation can change without breaking callers — that's encapsulation in action.
  • The reference-type trap is a classic gotcha. Show it live: two variables, modify through one, observe the other changed.
  • Use the Car or BankAccount example to draw a memory diagram on the board: stack holds the reference (address), heap holds the object.
  • Students often ask "when should I use a static method?" — answer: when the method does not depend on the state of any particular instance.

Common Mistakes

Forgetting new
Student s; // declared but NOT created — s is null
s.Name = "Dias"; // NullReferenceException

// Fix:
Student s = new Student();
s.Name = "Dias";
Making fields public instead of using properties
// Bad — no protection
public int Age;

// Good — controlled access
public int Age { get; set; }
Calling a static member on an instance
Counter c = new Counter();
c.GetTotal();         // ERROR — GetTotal is static
Counter.GetTotal();   // Correct
Confusing assignment with copying
Student original = new Student("Aisha", 20, "IT");
Student copy = original; // NOT a copy — same object!
copy.Name = "Bekzod";
Console.WriteLine(original.Name); // Bekzod — original changed too!
