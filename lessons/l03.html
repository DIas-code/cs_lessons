Part 1 — C# Basics

L03 — Console I/O + Operations

20 February 2026 · Friday · 80 min Done

We learn how to interact with the user through the console — reading input and writing output. Then we cover all arithmetic, assignment, and increment operators, and the powerful Math class from the BCL.

🎯 Learning Objectives

  • Read user input from the console using Console.ReadLine()
  • Convert string input to numeric types using Parse and Convert
  • Apply arithmetic operators and understand integer vs floating-point division
  • Use increment, decrement, and compound assignment operators
  • Apply common Math class methods
  • Understand operator precedence and use parentheses to control evaluation order

📖 Theory

1. Reading Input from the Console

User input is captured via the Console class. The most common method is Console.ReadLine(), which reads a full line of text and returns it as a string.

  • Console.ReadLine() — reads a line of text. Returns string? (nullable string). Always returns the input as a string regardless of what the user typed.
  • Console.Read() — reads a single character and returns its Unicode integer code. Rarely used.
  • Console.ReadKey() — reads a single key press. Returns a ConsoleKeyInfo struct. Useful for "press any key to continue" prompts.
ReadLine always returns a string! If the user types 42, Console.ReadLine() gives you the string "42", not the number 42. You must convert it before doing math.

2. Parsing User Input

Converting user input (string) to a number is one of the most common operations in console programs. You have three main approaches:

  • int.Parse(str) — converts a string to int. Throws FormatException if the input is not a valid number (e.g., user types "abc").
  • Convert.ToInt32(str) — same result, slightly more lenient. Also throws on invalid input.
  • int.TryParse(str, out int result) — safe version. Returns true if conversion succeeded, false otherwise. Never throws. Use this when user input cannot be trusted (which is always).

Equivalents exist for other types: double.Parse, decimal.Parse, etc.

3. Arithmetic Operators

OperatorNameExampleResult
+Addition10 + 313
-Subtraction10 - 37
*Multiplication10 * 330
/Division10 / 33 (integer!) or 3.333... (double)
%Modulo (remainder)10 % 31
Integer division — a very common bug. When both operands are int, the result is also int — the decimal part is truncated, not rounded. 10 / 3 = 3, not 3.33. Fix: make at least one operand a double — 10.0 / 3 or (double)10 / 3.

Modulo (%) gives the remainder after division. Uses:

  • Check if a number is even: n % 2 == 0
  • Check if divisible by 3: n % 3 == 0
  • Wrap a counter in a circular range: index % arrayLength

7. Operator Precedence

C# evaluates expressions following standard mathematical precedence (like BODMAS/PEMDAS):

  1. Parentheses () — highest priority
  2. Unary: ++, --, - (negation)
  3. Multiplicative: *, /, %
  4. Additive: +, -

When in doubt, use parentheses. They make code clearer and prevent mistakes.

Example: 2 + 3 * 4 = 14, but (2 + 3) * 4 = 20.

💻 Code Examples

C# — Reading and Parsing Input Program.cs 
// Ask the user for their name and age, then greet them

Console.Write("Enter your name: ");
string name = Console.ReadLine()!;   // ! tells compiler "trust me, not null"

Console.Write("Enter your age: ");
string ageInput = Console.ReadLine()!;
int age = int.Parse(ageInput);       // convert string → int

Console.WriteLine($"Hello, {name}! You are {age} years old.");
Console.WriteLine($"In 10 years you will be {age + 10}.");
C# — Arithmetic + Integer Division Demo Program.cs 
int a = 17;
int b = 5;

Console.WriteLine($"a + b = {a + b}");      // 22
Console.WriteLine($"a - b = {a - b}");      // 12
Console.WriteLine($"a * b = {a * b}");      // 85
Console.WriteLine($"a / b = {a / b}");      // 3  ← integer division!
Console.WriteLine($"a % b = {a % b}");      // 2  ← remainder

// Fix integer division by casting to double
double exactDiv = (double)a / b;
Console.WriteLine($"exact: {exactDiv}");     // 3.4

// Modulo use-case: check even/odd
Console.WriteLine(a % 2 == 0 ? "even" : "odd");  // odd
C# — Compound Assignment + Increment Program.cs 
int score = 10;

score += 5;   // score = 15
score *= 2;   // score = 30
score -= 8;   // score = 22
score++;      // score = 23
score--;      // score = 22

Console.WriteLine(score);   // 22

// Pre vs post increment — the difference
int x = 5;
Console.WriteLine(++x);   // 6  (incremented BEFORE printing)
Console.WriteLine(x++);   // 6  (printed first, THEN incremented)
Console.WriteLine(x);     // 7  (now it is 7)
C# — Math Class Program.cs 
Console.WriteLine(Math.Abs(-42));           // 42
Console.WriteLine(Math.Pow(2, 8));           // 256
Console.WriteLine(Math.Sqrt(144));          // 12
Console.WriteLine(Math.Max(17, 42));        // 42
Console.WriteLine(Math.Round(3.5));         // 4  (banker's rounding: rounds to even)
Console.WriteLine(Math.Floor(3.9));         // 3
Console.WriteLine(Math.Ceiling(3.1));       // 4

// Hypotenuse of a right triangle (a=3, b=4 → c=5)
double hyp = Math.Sqrt(Math.Pow(3, 2) + Math.Pow(4, 2));
Console.WriteLine($"Hypotenuse: {hyp}");    // 5
C# — TryParse (Safe Input Reading) Program.cs 
Console.Write("Enter a number: ");
string input = Console.ReadLine()!;

if (int.TryParse(input, out int number))
{
    Console.WriteLine($"You entered: {number}. Double: {number * 2}");
}
else
{
    Console.WriteLine("That is not a valid number.");
}

✏️ Practice Tasks

Task 1 Simple Calculator Easy

Write a program that reads two integers from the user and prints the result of all five arithmetic operations: +, -, *, /, %.

  • Use Console.Write() to prompt the user
  • Use int.Parse() to convert the input
  • For division, also print the exact decimal result (as a double)
Task 2 Rectangle Calculator Easy

Read the width and height of a rectangle (as double) from the user and print:

  • Area = width × height
  • Perimeter = 2 × (width + height)
  • Diagonal = √(width² + height²) — use Math.Sqrt and Math.Pow

Format all results to 2 decimal places.

Task 3 Time Breakdown Medium

Read a total number of seconds from the user and break it down into hours, minutes, and remaining seconds.

  • Hours: totalSeconds / 3600
  • Minutes: (totalSeconds % 3600) / 60
  • Seconds: totalSeconds % 60
  • Print: 3723 seconds = 1 hour(s), 2 minute(s), 3 second(s)

Tip: Use integer division and modulo. No need for Math here.

⚠️ Common Mistakes

Concatenating strings instead of adding numbers
string a = Console.ReadLine()!; string b = Console.ReadLine()!; Console.WriteLine(a + b); If you enter 3 and 4, this prints "34", not 7. You must parse first: int.Parse(a) + int.Parse(b).
Forgetting integer division
int result = 7 / 2; gives 3, not 3.5. This is especially tricky when both variables look like they should give a decimal answer. Always check: are both operands int? Cast one to double if you need the fraction.
Math.Pow returns double, not int
int result = Math.Pow(2, 8); is a compile error — Math.Pow always returns a double. Write double result = Math.Pow(2, 8); or cast: int result = (int)Math.Pow(2, 8);.
Not checking the result of TryParse
int.TryParse(input, out int n); — ignoring the return value means you proceed even if parsing failed, and n will be 0 (the default). Always use an if to check the return value.

🎓 Instructor Notes

⚡ Teaching tips
  • Live demo the integer division bug first. Ask students: "What does int x = 7 / 2 print?" Let them guess, then run it. The surprise of seeing 3 instead of 3.5 makes it memorable.
  • Walk through pre/post increment carefully. This is a classic interview question. Write int x=5; Console.WriteLine(x++); and ask: "what prints?" Many students expect 6. Run it, then explain why.
  • Introduce TryParse now, even briefly. Students immediately ask "what if the user types letters?" — answer it here. The full error-handling picture comes later, but TryParse is a simple tool they can use right away.
  • Operator precedence — use the calculator analogy. "Your phone calculator follows the same rules." Write 2 + 3 * 4 on the board and ask for answers — someone will say 20, someone 14. This makes the lesson concrete.
