Part 1 — C# Basics

L01 — C# Intro + Program Structure

06 February 2026 · Friday · 80 min Done

Introduction to the C# programming language and the .NET ecosystem. We set up the development environment, write our first console application, and learn how a C# program is structured from top to bottom.

🎯 Learning Objectives

  • Understand what C# is, where it runs, and what it is used for in the real world
  • Explain the role of the .NET runtime (CLR) and why it matters
  • Install Visual Studio and create a new Console Application project
  • Write, build, and run a first C# program
  • Read and explain every part of a basic C# program structure

📖 Theory

1. What is C#?

C# (pronounced "C sharp") is a modern, object-oriented, strongly typed programming language created by Microsoft. It was designed by Anders Hejlsberg and first released in 2000 as part of the .NET platform.

C# is used in a very wide range of applications:

  • Desktop apps — Windows Forms, WPF, .NET MAUI
  • Web backends — ASP.NET Core (APIs, MVC websites)
  • Mobile apps — .NET MAUI (iOS, Android, Windows)
  • Games — Unity (the most popular game engine uses C# as its scripting language)
  • Cloud services — Azure Functions, microservices
Why learn C#? C# is consistently in the top 5 most used programming languages worldwide (TIOBE, Stack Overflow surveys). It is in high demand for enterprise software, game development, and mobile applications.

2. The .NET Ecosystem

C# does not run alone — it runs on top of a platform called .NET. Think of .NET as the engine that powers your C# code.

The key components of .NET you need to know:

  • CLR — Common Language Runtime. The virtual machine that executes your program. Your C# source code is first compiled into Intermediate Language (IL), and the CLR then compiles IL into native machine code at runtime (this is called JIT — Just-In-Time compilation).
  • BCL — Base Class Library. A huge collection of ready-to-use classes: Console, Math, List<T>, file I/O, networking, etc. You do not write these from scratch.
  • SDK — Software Development Kit. The tools you install to write and build .NET applications: compilers, CLI tools, project templates.
Brief history — don't confuse these:
  • .NET Framework (2002–2019) — Windows-only, legacy
  • .NET Core (2016–2020) — cross-platform rewrite
  • .NET 5 – 10 — the current unified platform (cross-platform, open source). Even-numbered releases are LTS.
In this course we use .NET 10 (LTS — Long Term Support).

3. Setting Up the Environment

We use Visual Studio 2026 Community — it is free, full-featured, and the industry standard for C# and .NET MAUI development on Windows.

Installation steps:

  1. Go to visualstudio.microsoft.com and download Visual Studio 2026 Community
  2. Run the installer and select the workload: .NET desktop development
  3. For MAUI (later in the course), also select: .NET Multi-platform App UI development
  4. Click Install. This may take 10–30 minutes depending on your connection.
  5. Launch Visual Studio → Sign in (or skip) → Start
Minimum system requirements: Windows 10/11, 4 GB RAM (8 GB recommended), 10 GB free disk space. For Android MAUI emulator: 16 GB RAM recommended.

4. Creating Your First Project

  1. Open Visual Studio → Create a new project
  2. Search for Console App (make sure it says C#, not VB)
  3. Click Next
  4. Name your project (e.g., HelloWorld), choose a folder, click Next
  5. Select framework: .NET 10 (Long Term Support)
  6. Click Create

Visual Studio creates the project and opens Program.cs — your entry point.

5. Understanding Project Structure

After creating a Console App project, you will see:

  • Program.cs — the main source file where your code lives. In modern C# (v9+), this uses top-level statements — no boilerplate needed.
  • ProjectName.csproj — the project configuration file (XML format). It tells .NET which SDK version to use, target framework, dependencies, etc.
  • bin/ — compiled output (the executable your computer actually runs). You do not edit anything here.
  • obj/ — intermediate build files. Ignore this folder.

6. Your First C# Program

Modern C# (version 9 and above) supports top-level statements. This means you can write executable code directly in Program.cs without wrapping it in a class or a Main method.

Behind the scenes, the compiler generates the class and Main method for you. This is purely syntactic sugar — both styles produce identical compiled output.

7. Anatomy of a Traditional C# Program

It is important to also understand the "classic" structure, because you will encounter it in older codebases and tutorials:

  • using directives — import namespaces (like Python's import)
  • namespace — groups your code logically (prevents name conflicts)
  • class — every C# program has at least one class
  • static void Main() — the entry point; this is where execution starts

8. Comments in C#

Comments are notes in your code that the compiler completely ignores:

  • // single-line comment
  • /* multi-line comment */
  • /// XML doc comment — used to generate documentation (advanced)

💻 Code Examples

Open Visual Studio, create a Console App project, and try each example below. Run with Ctrl+F5.

C# — Top-Level Statements (Modern) Program.cs 
// Example 1 — Hello World (C# 9+ style)
// No class, no Main() needed — just write code directly

Console.WriteLine("Hello, World!");
Console.WriteLine("Welcome to C#!");
Console.WriteLine("My first program.");
C# — Traditional Structure (Classic) Program.cs 
// Example 2 — The same program, written in the traditional style
// This is what ALL C# code looked like before version 9

using System;         // import the System namespace

namespace HelloWorld  // namespace groups related classes
{
    class Program       // every C# program has at least one class
    {
        static void Main(string[] args)  // entry point
        {
            Console.WriteLine("Hello, World!");
            Console.WriteLine("Welcome to C#!");
            Console.WriteLine("My first program.");
        }
    }
}
C# — Console Output Methods Program.cs 
// Example 3 — Different ways to write to the console

// WriteLine — prints text and moves to the NEXT LINE
Console.WriteLine("This is on line 1.");
Console.WriteLine("This is on line 2.");

// Write — prints text WITHOUT moving to next line
Console.Write("Hello, ");
Console.Write("World!");   // continues on the same line
Console.WriteLine();        // now move to next line (empty WriteLine)

// String interpolation — embed variables into strings (preview)
string name = "Alice";
int age = 21;
Console.WriteLine($"Name: {name}, Age: {age}");

// Output:
// Name: Alice, Age: 21
C# — Comments Program.cs 
// Single-line comment — anything after // is ignored by the compiler

Console.WriteLine("Hello!"); // inline comment at end of line

/*
   Multi-line comment.
   Everything between the slash-star and star-slash is ignored.
   Useful for longer explanations or temporarily disabling code.
*/

// Console.WriteLine("This line is commented out — it won't run!");

Console.WriteLine("Done.");

⚠️ Common Mistakes

Forgetting the semicolon
Every statement in C# must end with ;. Writing Console.WriteLine("Hello") without the semicolon causes a compile error. C# is not Python — whitespace and line endings have no meaning; the semicolon is the statement terminator.
Case sensitivity — console vs Console
C# is case-sensitive. Console is the class name and must start with a capital C. Writing console.writeline() will not compile. Similarly, Main, String, WriteLine — all capitalised.
Forgetting to use Console.ReadLine() to pause the console
When running with Ctrl+F5 (Start Without Debugging), the console stays open. But if you run with F5 (Debug) and your program ends immediately, the window closes. Add Console.ReadLine(); at the end to pause and wait for the user to press Enter. (Not always needed — depends on project settings.)
Mixing up Write and WriteLine
Console.Write() prints text and stays on the same line. Console.WriteLine() prints text and moves to the next line. Students often use Write everywhere and wonder why their output runs together.

🎓 Instructor Notes

⚡ Teaching tips for this lesson
  • Budget 15–20 min for installation. Many students arrive without Visual Studio 2026 installed. Walk through it live on the projector while students follow along. Have a USB drive with the offline installer as backup.
  • Show BOTH styles of Hello World (top-level + traditional), then explain which one you'll use going forward and why. Students who look up external tutorials will see the traditional style, so they must recognise it.
  • Demonstrate Ctrl+F5 vs F5. Many students accidentally close the console window immediately. Establishing the keyboard shortcut habit early prevents confusion.
  • Explain IL briefly. Showing students that C# compiles to IL (not machine code directly) helps them understand why .NET runs on multiple platforms. A one-sentence analogy works: "IL is like a universal language that the CLR on each platform understands."
  • Do not go deep on namespaces and classes today. Just show the structure visually — these will be covered properly in the OOP lessons.
